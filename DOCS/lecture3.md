# Лекция. Распределённые базы данных (РБД).
### date: 19.05.26

## Ввдение

РБД - совокупность логически связанных данных, физически размещённых на нескольких устройствах

## Архитектура РБД

### Шардирование данных
 - разделение данных на части (шарды), каждый шард на отдельном сервере

### Репликация данных
Репликация - создание и хранение копий данных на нескольких узлах

Цель - обеспечить работу системы даже при отказе одного сервера

## Основные проблемы 
- Сетевые задержки
### CAP-теорема 
- РБД не способна соответствовать 3-ём одновременно:
    - Consistency - Согласованность
    - Availability - Доступность
    - Partition Tolerance - Устойчивость к разделению

### Согласованность данных и балансировка нагрузки

Согласованность - при репликации изменения передаются с некоторой задержкой - следовательно, разные серверы имеют разные версии данных

Балансировка нагрузки - распределение запросов между серверами


### Масштабируемость и сложность администрирования

Масштабируемость - 

Сложность администрирования - из-за множества связей между серверами становится затруднительно, в отличие от централизованной системы.
